# story_edit Checklist

- Source story file exists and is readable.
- User directives are split into atomic change requests.
- Each request is classified (`mandatory`, `optional`, `conflict-risk`).
- Rewrite scope is confirmed before editing.
- Revised story has no timeline or motivation contradictions.
- Edit log records accepted and rejected directives with reasons.
- Revised story and edit log are both saved with metadata block.
